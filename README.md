Here is the project you are going to work on during this week starting from today, along with the tasks you have to do during that time. 

**Personal Spending Tracker**
**Description:** Build a web page where users track their daily spending. Each entry includes an amount, a category (for example food, transport, entertainment), and a date entered by the user. The page shows all spending entries, calculates and displays total spending per category, and shows how spending changes over time using a simple chart. The page automatically highlights unusually large expenses so they stand out. Users can add, edit, and delete entries. All data must be stored in the browser so that refreshing the page does not remove it.

**Tools to use:**
- JavaScript (ES6+)
- HTML
- CSS
- LocalStorage
- Jest or Vitest
- Chart.js

**Tasks for week 1:**
1. Design and implement a data structure for a spending entry that includes a numeric value, a category name, a date or timestamp, and a unique identifier.

2. Build functionality that allows users to add new spending entries by entering the value, category, and date.

3. Implement the ability to edit existing entries and update their value, category, or date.

4. Implement the ability to delete entries and ensure they are removed from both the interface and stored data.

5. Persist all entries using LocalStorage so spending data remains available after refreshing the page.

6. Implement logic to calculate total spending grouped by category and keep these totals up to date as entries change.

7. Implement logic to calculate totals over fixed time ranges (for example per day or per week).

8. Implement logic to detect entries that are unusually high or low compared to recent values and mark them clearly.

9. Display calculated summaries in a clear textual format and visualize at least one spending trend using a chart.

10. Create a README file that explains how to run the project locally and includes 2–3 keywords so another team can guess what the project is about.
